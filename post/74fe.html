requests模块

pip install requests(第三方扩展包)

response.content: 字符串

response.content.decode(): 二进制

字符串==>二进制 str.encode()

二进制==>字符串 bytes.decode()

response.content.decode(‘utf-8’)

response.text是猜测来解码

response.content.decode()可以根据自己设置来解码

xpath表达式

如果想提取数据, 可以通过xpath, 但是得将源代码转换成xml格式, 转换完了, 就可以通过xpath表

达式来写

pip install lxml(如果有红色波浪线没问题)

表达式 描述
/ 从根节点选取
// 从匹配选择的当前节点选择文档的节点,而不考虑他们的位置
. 选择当前节点
.. 选择当前节点的父节点
@ 选取属性,例如://div[@class=”bule”]
//div[1] 选择div的第一个
//div[last()] 选择最后一个div
//div[last()-1] 选择倒数第二个
//div[position<3] 选择前两个div
//div[@class=”box”] 选择class属性值为box的标签
//div[@price>20] 选择price属性值大与20的标签
* 通配符
//div|//span 所有的div和span标签
div[contains(@class,’swiper-slide’)] div里面class内容里包含,’swiper-slide’
/img/@src 获取img标签内src的属性值
/div/text() 获取div标签下text文本

pyquery

同xpath, 能够将源代码转换成pyquery对象, 然后再通过类jquery选择器来获取内容

语法 结果
doc(‘xxx’) 里面写jquery语法
doc.find(‘li’) doc里面所有的li标签
doc.children(‘li’) doc里面所有儿子节点li
doc.parent() doc的父亲节点
doc.parents doc的所有祖宗节点
doc.sliblings 所有兄弟节点
attr(‘href’)或者a.attr.href 获取属性
a.text() 获取文本
a.html() 获取a里面所有

正则

  • 同以上两种方法, 另外两种方法是将源代码转换成另外的数据类型然后再通过自身的语法来提取
  • 正则, 直接针对字符串来提取内容
  • xpath, pyquery针对结构化的数据非常好用, 但是如果结构混乱则不好用
  • re针对任何样式的数据都能够用, 但是不管针对任何样式也都比较麻烦
  • 总结**: 能够使用xpath则会使用它, 除非用不了, 再用re**

内置模块, 不需要安装

  • re.match(‘表达式’, 提取源内容)

    从源内容的头部内容开始, 根据表达式的规则来找

    正则类型, 只能得到一个, 通过group()得到最终数据

  • re.search(‘表达式’, 提取源内容)

    源内容的任何地方, 都可以根据表达式的规则来找

    正则类型, 只能得到一个, 通过group()得到最终数据

  • re.fifindall(‘表达式’, 提取源内容)

    源内容的任何地方, 都可以根据表达式的规则来找

    列表, 所有匹配到的都拿出来

re模块

  1. ‘\d’匹配0-9之间的数值
  2. ‘+’重复前一个匹配字符一个或者多个匹配字符
  3. ‘*’重复前一个匹配字符零次或者多次
  4. ‘?’重复前一个匹配字符零次或者一次
  5. ‘.’代表任何一个字符,并未声明不代表字符’\n’
  6. ‘ | ‘代表把左右分为两个部分,要么是….要么是…..
  7. 特殊符号使用反斜线 ‘ \ ‘引导,例如 ‘ \r’,’ \n ‘,’\t’
  8. “[ ]”中的字符任选择一个 :列如[0-9],[A-Z][a-z]
  9. ‘\b’表示单词结尾,包括空白字符或字符串结尾
  10. “ ^ “出现在 [ ] 第一个字符串前面,代表取反,列如: ‘[^0-9 ]’
  11. ‘\s’匹配任何字符串,等价于”[\r \n \x20 \t \f \v]”
  12. ‘w’匹配包括下划线子内单词字符,等价于’[0-9 A-Z a-z]’
  13. ‘$’字符匹配字符串结尾位置
  14. 使用’( )’把里面看作一个整体
